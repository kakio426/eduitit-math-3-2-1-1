# 매스몬 검산 자물쇠 설명 보고서

## 1. 개요

`매스몬 검산 자물쇠`는 3학년 2학기 2단원 4차시에서 다루는 곱셈을 이용한 나눗셈 검산을 게임 흐름으로 연습하는 에듀잇티 수학 게임입니다. 학생은 나눗셈 결과의 몫과 나머지를 사용해 원래 나누어지는 수가 맞는지 확인합니다.

핵심 목표는 `몫 × 나누는 수 + 나머지 = 나누어지는 수` 관계를 눈에 보이는 자물쇠 계산판으로 반복하게 만드는 것입니다.

## 2. 학습 설계

- 문제 유형: 나머지가 있는 두 자리 수 ÷ 한 자리 수의 검산
- 문제 은행: 나누는 수 2~9, 몫 4~18, 나머지 1~나누는 수보다 작은 수로 생성
- 라운드 길이: 10문제
- 입력 방식: 곱하기, 나머지 더하기, 원래 수 확인 3단계 4지선다 선택
- 1단계: `몫 × 나누는 수` 값을 골라 곱한 값을 만든다.
- 2단계: 곱한 값에 `나머지`를 더해 원래 수를 복원한다.
- 3단계: `검산 결과 = 원래 수` 비교식 중 맞는 식을 고른다.
- 오답 설계: `몫 × 나누는 수`에서 멈추는 나머지 더하기 누락 오답을 2단계와 3단계 보기에 포함한다.
- 보상: 한 문제 완료마다 금고 해제 이벤트를 1회 적용한다.
- 결과 등급: 금고 해제 에너지와 정답 수 게이트를 함께 사용해 자물쇠, 금고, 대형금고, 비밀금고, 보물고 중 하나를 공개한다.
- 비밀 등급: 무지개 코어를 얻으면 전설 금고가 열린다.
- 최종 보상: 도달한 금고 등급 자체가 보상이며, 매스몬 도감 수집 구조는 사용하지 않는다.

### 교육적 의도

검산은 답을 다시 맞혀 보는 절차가 아니라, 나눗셈의 구조를 확인하는 방법입니다. 이 게임은 `47 ÷ 5 = 9 ... 2`를 `9 × 5 = 45`, `45 + 2 = 47`로 나누어 보여 줍니다. 특히 나머지를 더하지 않고 `45`에서 멈추는 선택지를 반복적으로 넣어, 검산식의 마지막 `+ 나머지`가 왜 필요한지 행동으로 확인하게 했습니다.

보상은 `금고 해제 에너지` 하나로 단일화했습니다. 정답을 많이 맞히면 분명히 유리하지만, 보상 이벤트에는 증가, 감소, 다시 잠김, 마스터키, 무지개 코어가 섞여 있어 매 판 결과가 달라집니다. 틀렸던 문제는 오류 처리 이벤트만 적용해 일부러 틀리는 전략을 막습니다. 플레이 중에는 도달 금고 등급을 미리 강조하지 않고, 금고 주변 빛과 잠금등만 에너지에 따라 켜지게 했습니다.

## 3. 게임 흐름

```text
첫 화면 -> 설명 화면 -> 곱하기 -> 나머지 더하기 -> 원래 수 확인 -> 금고 해제 이벤트 -> 다음 문제 또는 결과 -> 금고 등급 공개
```

학생은 먼저 몫과 나누는 수를 곱합니다. 다음에는 곱한 값에 나머지를 더합니다. 마지막으로 `47 = 47`, `45 = 47`처럼 검산 결과와 원래 수를 비교한 식을 보고 맞는 식을 고릅니다.

## 4. 화면별 설명

### 첫 화면

첫 화면은 `cover-generated.webp`를 RasterStage 배경으로 사용합니다. 금고와 다이얼, 매스몬 동행 캐릭터가 보이는 보안실 장면 위에 게임 제목, 한 줄 목표, 시작 버튼을 HTML로 얹습니다. 생성 이미지에는 고정 텍스트를 넣지 않아 제목과 버튼이 선명하게 유지됩니다.

### 설명 화면

설명 화면은 3단계만 보여 줍니다.

1. 몫과 나누는 수를 곱해요.
2. 곱한 값에 나머지를 더해요.
3. 원래 수와 맞으면 자물쇠가 열려요.

버튼 문구는 다음 행동이 바로 보이도록 `자물쇠 풀기`로 두었습니다.

### 문제 화면

문제 화면은 왼쪽에 보안실 배경과 금고 해제 에너지, 오른쪽에 나눗셈 결과와 검산 계산판을 둡니다. 문제와 선택지가 가장 크게 보이도록 배경 이미지는 보조 역할로 제한했습니다.

검산 계산판은 1단계 곱하기, 2단계 나머지 더하기, 3단계 원래 수 확인 칸으로 구성됩니다. 3단계 선택지는 숫자를 한 번 더 누르는 방식이 아니라 `검산 결과 = 원래 수` 비교식으로 보여 줍니다. 각 단계가 끝나면 해당 칸이 채워지고, 다음 칸이 강조됩니다.

왼쪽 보안실 영역은 플레이 중 금고 등급을 미리 공개하지 않습니다. 대신 해제 에너지에 따라 금고 주변 빛, 다이얼, 잠금등이 점점 켜집니다. 최종 등급 강조는 결과 화면 측정 뒤에만 표시됩니다.

### 보상 화면

한 문제의 3단계 계산이 끝나면 화면 중앙에 금고 해제 이벤트가 뜹니다. 정답 문제는 보안 해제, 오류 경고, 강화 열쇠, 마스터키, 다시 잠김, 무지개 코어 중 하나가 나오며, 문제 안에서 한 번이라도 틀리면 오류 처리만 적용됩니다. 보상은 금고 해제 에너지 하나로만 적용됩니다. 무지개 코어가 나오면 보안실 미터와 금고 빛도 무지개 상태로 바뀝니다.

### 결과 화면

결과 화면은 도달 등급별 RasterStage 배경을 사용합니다. 자물쇠, 금고, 대형금고, 비밀금고, 보물고, 전설 금고 이미지가 따로 있으며, 생성 이미지에는 점수·정답 수·버튼을 넣지 않았습니다. 그 위에 해제 에너지 측정 막대, 정답 수, 도달 등급, 칭찬 문구, 다시하기 버튼을 HTML로 얹습니다.

### 다시하기 결과 화면

해제 에너지가 부족하거나 정답 수가 부족하면 `result-retry-generated.webp`를 RasterStage 배경으로 사용합니다. 닫힌 금고와 다시 도전 문구를 보여 줍니다.

## 5. 매스몬 역할

이 차시에서 매스몬은 금고 보안실을 함께 살피는 동행 캐릭터로 등장합니다. 학생이 얻는 중심 보상은 `금고 등급 도달`이며, 도감 수집 구조는 사용하지 않습니다.

## 7. 검증 기록

- `node --check`로 HTML 내부 스크립트 문법을 확인했습니다.
- VM 테스트로 문제 생성 항등식, 대표 오답 포함, 보상 이벤트 분기, 10문제 완주 흐름을 확인했습니다.
- Chrome headless로 첫 화면, 설명, 문제, 보상, 오류 보상, 성공 결과, 재도전 결과 화면을 캡처했습니다.
- 문제 생성 규칙은 나머지가 항상 1 이상이고 나누는 수보다 작도록 구성했습니다.
- 2단계와 3단계 선택지에는 나머지 더하기를 빠뜨린 오답이 포함됩니다.
- 프로덕션 HTML의 화면 건너뛰기 훅을 제거했습니다.
- 플레이 중 금고 등급은 강조하지 않고, 결과 화면에서만 도달 등급을 공개합니다.
- 결과는 금고 등급 하나로 끝나며, 별도 도감이나 누적 저장 구조를 만들지 않았습니다.
