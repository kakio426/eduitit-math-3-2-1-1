# 매스몬 별 줍기 설명 보고서

## 1. 개요

- 차시: 3학년 2학기 2단원 3차시
- 배움주제: 나머지가 있는 나눗셈
- 한 줄 목표: "똑같이 나누고 남는 별을 주워 보자!"
- 성취기준: `[4수01-06]` 안내용 표기이며, 문서 확정 전 교육부 고시 원문 대조가 필요합니다.

이 차시는 `문제 풀이 -> 별빛 보상 -> 별 등급 결과` 흐름으로 설계했습니다. 계산은 별을 줍기 위한 행동 도구이고, 보상은 유성우와 구름처럼 무작위로 변해 다시 도전하고 싶은 긴장감을 만듭니다. 최종 별빛 숫자는 결과 전까지 공개하지 않습니다.

## 2. 학습 설계

- 문제 유형: `(몇십몇) ÷ (몇) = 몫 ... 나머지`
- 생성 규칙: 나누는 수 3~9, 나머지 1~(나누는 수-1), 나머지 0 제외
- 라운드 길이: 10문제
- 입력 방식: 1단계 몫 고르기, 2단계 나머지 고르기, 3단계 검산 자동 확인
- 대표 오개념: 몫을 하나 작게 골라 `나머지 >= 나누는 수`가 되는 보기, 나머지를 `정답 + 나누는 수`로 고르는 보기
- 피드백: 첫 오답은 힌트, 두 번째 오답은 정답 공개 후 다음 단계 진행

## 3. 보상 구조

중심 보상은 `별빛` 하나입니다. 정답을 맞히면 랜덤 이벤트가 한 번 적용되고, 정답으로 확인한 나머지 별 수가 결과에서 보너스 별빛으로 더해집니다. 오답이나 정답 공개로 끝난 문제의 나머지는 확인 문구로만 보여 주며 등급 점수에는 더하지 않습니다.

- 증가: 유성우
- 큰 증가: 큰 유성우
- 감소: 구름
- 0: 깜깜
- 조기 마무리: 별똥별
- 최고 직행: 무지개 별
- 중간 오답: 구름 처리

결과 등급은 `별먼지 -> 작은별 -> 별자리 -> 은하 -> 대우주`와 전설 등급 `무지개 초신성`입니다. 별빛 점수만 높아도 정답 수 게이트를 통과해야 높은 등급에 닿도록 했고, 기본 `별먼지` 등급과 위로 피드백으로 서툰 학생도 빈손으로 끝나지 않게 했습니다.

## 4. 화면별 설명

### 첫 화면

`cover-generated.webp`를 RasterStage 배경으로 사용합니다. `에듀잇티 수학 게임` 브랜드 배지, 차시 배지, 배움주제 배지, 제목, 한 줄 목표, `시작` 버튼은 HTML 오버레이입니다.

### 설명 화면

`tutorial-generated.webp` 배경 위에 3단계만 표시합니다. 버튼은 다음 행동을 말하는 `별 줍기`입니다.

### 문제 화면

`board-night.webp` 배경 위에 전체 별 수, 한 묶음 수, 몫, 나머지, 검산식을 보여 줍니다. 몫과 나머지를 맞히기 전에는 남은 별을 시각적으로 드러내지 않아 답이 새지 않게 했습니다.

### 보상 화면

`star.webp`를 중심 오브젝트로 두고 CSS 클래스만 바꿔 유성우, 구름, 깜깜, 별똥별, 무지개 별을 표현합니다. 보상은 한 문제당 한 번만 적용됩니다.

### 결과 화면

`result-stage.webp`를 배경으로 사용하고, 도달 등급에 맞는 `result-star-dust.webp`, `result-star-small.webp`, `result-star-constellation.webp`, `result-star-galaxy.webp`, `result-star-universe.webp`, `result-star-rainbow.webp` 중 하나를 표시합니다. 최종 별빛, 정답 수, 등급명, 다시하기 버튼은 HTML 오버레이입니다. 매스몬 동행 캐릭터는 `_shared/mathmon/base-pack/`의 `mathmon-8-unicornmon.webp` 배포본을 사용합니다.

## 5. 검증 포인트

- 나머지는 항상 `1 <= 나머지 < 나누는 수`
- 보기에는 `나머지 >= 나누는 수` 오개념이 포함됨
- 첫 화면 3요소가 가장 먼저 보임
- 문제와 선택지가 보상보다 우선함
- 결과는 차시 자체 완결형 등급으로 끝남
- 학생 화면에 내부 작업실 이름을 노출하지 않음
