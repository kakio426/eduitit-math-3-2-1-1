# 매스몬 차시별 제작 프롬프트 뱅크

이 문서는 `ai mart` 작업실에서 차시를 만들거나 크게 고칠 때 Codex와 Claude에 붙여 넣는 프롬프트 모음이다. 목적은 문제 화면이 설명판으로 가득 차는 것을 막고, 초등학교 3학년 학생이 수학 원리를 **보이는 조작**으로 이해하며 풀게 만드는 것이다.

## 사용 방법

1. 새 차시를 만들거나 기존 차시 문제 화면을 고칠 때 먼저 아래 **공통 제작 프롬프트**를 붙인다.
2. 이어서 해당 차시의 **차시별 추가 프롬프트**를 붙인다.
3. 결과 스크린샷을 보고 3초 안에 학생이 할 행동을 말할 수 없으면 실패로 보고 다시 줄인다.

## 공통 제작 프롬프트

```text
너는 /Users/yubyeongju/ai mart 작업실의 에듀잇티 매스몬 수학 게임 차시를 만드는 에이전트다.

반드시 먼저 읽고 따른다:
- AGENTS.md
- CLAUDE.md
- SERIES_CONTRACT.md
- .claude/skills/eduitit-mathmon-lesson/SKILL.md
- .claude/skills/eduitit-mathmon-lesson/references/effect-design.md
- .claude/skills/eduitit-mathmon-lesson/references/verification.md

이번 작업의 최우선 목표는 초등학교 3학년 학생이 수학 원리를 직관적으로 보고 직접 고르게 만드는 것이다.

문제 화면 강제 규칙:
1. 학생이 3초 안에 "지금 무엇을 고르는지" 말할 수 있어야 한다.
2. 기본 노출 요소는 큰 문제 / 현재 단계 조작판 / 한 줄 지시문 / 선택지 4개뿐이다.
3. 개념 설명판, 풀이 해설판, 힌트 내용, 계산 미리보기, 보상 상태판을 한 화면에 동시에 펼치지 않는다.
4. 단계형 문제는 현재 단계만 크게 보여 준다. 이전 단계는 작은 완료 칩, 다음 단계는 ? 또는 잠금 상태로 둔다.
5. 힌트는 기본으로 닫는다. 열어도 지금 단계 힌트 1개만 보여 준다.
6. 원리는 긴 문장으로 설명하지 말고 블록, 칸, 스티커, 화살표, 자리값 묶음 같은 시각 조작으로 보여 준다.
7. 선택지는 문제 화면에서 가장 눈에 잘 띄는 한 덩어리로 둔다.
8. 효과는 단계 정답, 보상 이동, 중심 오브젝트 변화, 랜덤 이벤트 충격, 결과 측정처럼 학생 행동과 보상을 연결하는 곳에만 둔다.
9. 문제보다 패널이 많거나, 읽어야 할 문장이 많거나, 무엇을 눌러야 할지 바로 안 보이면 완성하지 말고 즉시 덜어낸다.

금지:
- 교사용 설명처럼 긴 문장
- 같은 원리를 왼쪽 설명판과 오른쪽 계산판에 중복 표시
- 전체 풀이 과정을 처음부터 모두 펼쳐 놓기
- 보상/진행도/장식이 문제와 선택지보다 눈에 띄게 만들기
- 별·코인·하트처럼 중심 보상과 별개인 효과를 새 보상 체계처럼 추가하기
- 학생 화면에 핵심 메커니즘, 메타 필드, 정답 수 게이트 같은 제작자 용어 쓰기

검증:
- 문제 화면 스크린샷을 보고 큰 문제, 현재 단계, 한 줄 지시, 선택지가 가장 먼저 보이는지 확인한다.
- "3초 행동" 한 문장으로 화면을 설명할 수 있어야 한다.
- 효과가 문제·선택지를 가리지 않고, 방금 한 수학 행동과 직접 연결되어 보이는지 확인한다.
- node scripts/check-stage-ratio.mjs를 통과시킨다.
```

```text
3초 행동: 세 자리 수와 한 자리 수를 곱한 답을 고른다.

문제 화면은 큰 곱셈식과 4개 선택지만 중심에 둔다. 상자 보상, 점수, 콤보는 작게 보조로 둔다.
원리 도움은 필요할 때만 자리값 칩(백/십/일)로 보여 준다. 긴 풀이 설명은 기본 화면에 펼치지 않는다.
```

```text
현재 자리(일/십/백)만 크게 보여 준다. 이전 자리는 완료 칩, 다음 자리는 ?로 접는다.
올림은 작은 화살표와 칩으로만 보여 주고, 전체 세로셈 해설판을 동시에 펼치지 않는다.
로켓 연료 상태는 작게 두고 선택지를 방해하지 않는다.
```

```text

3초 행동: 0을 잠깐 떼고 핵심 곱을 먼저 고른다. 다음 단계에서는 붙일 0 개수를 고른다.

1단계에는 핵심 곱 칸만 크게 보여 준다. 0 스티커 칸과 완성 칸은 ?로 둔다.
2단계에는 붙일 0 스티커만 크게 보여 준다. 이미 고른 핵심 곱은 작은 완료 칩으로 접는다.
생산량, 출하 등급, 매스몬 카드는 문제 화면에서 전면에 두지 않는다.
```

### 3-2-1-4 매스몬 로봇 합체

```text

3초 행동: 지금 부품에 필요한 부분곱 하나를 고른다.

부분곱 2개를 한꺼번에 큰 패널로 펼치지 않는다. 현재 곱할 자리만 크게 보여 준다.
완료된 부분곱은 작은 부품 칩으로 접고, 더하기 단계는 마지막에만 연다.
분해 원리는 로봇 부품 2칸과 합체 화살표로 보여 주고 긴 설명은 쓰지 않는다.
```

```text
게임명: 매스몬 물통 채우기 시합
배움주제: 들이 비교와 L, mL
3초 행동: 눈금이 가리키는 들이를 고른다.

물통 눈금을 가장 크게 보여 준다. 단위 설명판을 기본으로 펼치지 않는다.
선택지는 L/mL 값 후보로 크게 둔다.
비교 문제에서는 두 물통 중 더 많이 든 쪽만 고르게 한다.
```

### 3-2-5-2 매스몬 음료 제조 주문

```text
차시: 3학년 2학기 5단원 2차시
게임명: 매스몬 음료 제조 주문
배움주제: 들이의 덧셈, 뺄셈과 어림
3초 행동: 주문에 맞는 L와 mL 계산 결과를 고른다.

현재 단계는 mL끼리 계산 또는 L로 받아올림/내림 중 하나만 크게 보여 준다.
전체 세로 계산판과 단위 변환 설명을 동시에 펼치지 않는다.
1000mL가 1L로 바뀌는 것은 병 하나가 채워지는 시각 조작으로 보여 준다.
```

### 3-2-5-3 매스몬 저울 균형

```text
차시: 3학년 2학기 5단원 3차시
게임명: 매스몬 저울 균형
배움주제: 무게 비교와 kg, g, t
3초 행동: 저울을 맞출 무게 또는 더 무거운 쪽을 고른다.

저울 그림을 크게 보여 주고 선택지는 무게추 후보로 둔다.
단위 설명표를 펼치지 않는다.
kg/g 변환이 필요하면 1000g 묶음이 1kg 무게추로 바뀌는 모습으로 보여 준다.
```

### 3-2-5-4 매스몬 택배 무게 맞추기

```text
차시: 3학년 2학기 5단원 4차시
게임명: 매스몬 택배 무게 맞추기
배움주제: 무게의 덧셈, 뺄셈과 어림
3초 행동: 무게 한도에 맞는 계산 결과를 고른다.

현재 단계는 g 계산 또는 kg 받아올림/내림 중 하나만 크게 보여 준다.
택배 상자, 한도 게이지, 선택지를 중심에 둔다.
여러 상자의 세부 설명을 한 화면에 모두 펼치지 않는다.
```

## 6단원 자료의 정리

### 3-2-6-1 매스몬 인구조사

```text
차시: 3학년 2학기 6단원 1차시
게임명: 매스몬 인구조사
배움주제: 자료를 표로 정리하기
3초 행동: 자료를 세어 알맞은 표 칸 값을 고른다.

한 번에 한 항목만 세게 한다. 전체 자료와 전체 표를 동시에 복잡하게 펼치지 않는다.
현재 세는 항목은 색으로 강조하고, 표의 해당 칸만 크게 보여 준다.
선택지는 개수 후보로 둔다.
```

### 3-2-6-2 매스몬 그림 단위 해독

```text
차시: 3학년 2학기 6단원 2차시
게임명: 매스몬 그림 단위 해독
배움주제: 그림그래프 읽기
3초 행동: 그림 1개의 단위를 보고 값을 고른다.

현재 항목의 그림 줄만 크게 보여 준다. 전체 그래프 해설을 펼치지 않는다.
그림 1개가 몇을 뜻하는지는 작은 고정 배지로만 둔다.
선택지는 값 후보로 크게 둔다.
```

### 3-2-6-3 매스몬 그림 도장 찍기

```text
차시: 3학년 2학기 6단원 3차시
게임명: 매스몬 그림 도장 찍기
배움주제: 그림그래프로 나타내기
3초 행동: 값에 맞게 큰 그림과 작은 그림 수를 고른다.

현재 항목 하나만 그래프로 만든다.
큰 그림 몇 개, 작은 그림 몇 개를 단계별로 묻고 한 화면에 모든 항목을 펼치지 않는다.
도장 찍기 조작으로 원리를 보여 주고 긴 계산 설명은 숨긴다.
```

### 3-2-6-4 매스몬 데이터 탐정

```text
차시: 3학년 2학기 6단원 4차시
게임명: 매스몬 데이터 탐정
배움주제: 그래프 해석과 비교
3초 행동: 그래프를 보고 가장 큰 값, 가장 작은 값, 차이를 고른다.

한 문제에서는 비교 목표 하나만 묻는다. 가장 큰 값/작은 값/차이/합계를 동시에 묻지 않는다.
현재 비교할 막대나 그림 줄만 강조한다.
선택지는 결론 후보로 두고, 해설 문장은 결과 피드백에서 짧게 보여 준다.
```
