# 효과 설계 기준

매스몬 차시의 효과는 장식이 아니라 **학생의 수학 행동이 게임 세계를 바꿨다는 신호**다. 문제와 선택지를 가리거나, 보상 체계를 하나 더 만든 것처럼 보이면 실패로 본다.

## 효과를 주면 좋은 위치

1. **현재 단계 정답 직후**
   - 방금 고른 자리, 칸, 블록, 부품만 짧게 반짝인다.
   - 예: 세로셈 답 칸 금빛 펄스, 올림 칸 스파크, 나눗셈 바구니 체크, 분수 조각 맞물림.
   - 목적: "내가 방금 한 계산이 여기에 들어갔다"를 보이게 한다.

2. **문제 완료 후 보상으로 이동하는 순간**
   - 오른쪽 문제판에서 왼쪽/중앙 보상 오브젝트로 에너지, 빛, 물질, 부품이 이동한다.
   - 예: 로켓 연료 라인, 상자 점수 빛줄기, 농장 씨앗 날아가기, 로봇 부품 장착.
   - 목적: 계산이 보상으로 이어지는 행동의 도구라는 구조를 느끼게 한다.

3. **중심 보상 오브젝트 내부**
   - 보상은 물질감 있게 변화한다. 차오름, 찰랑임, 쌓임, 켜짐, 붙음, 열림처럼 한눈에 보이는 변화를 쓴다.
   - 예: 연료 액체 출렁임, 상자 등급 광택, 엘리베이터 층 불빛, 별 개수 채움, 자물쇠 핀 움직임.
   - 목적: 숫자보다 먼저 "이번 판이 변하고 있다"를 보이게 한다.

4. **랜덤 이벤트 충격**
   - 증가/대량/감소/0/즉시완성/특별 이벤트는 같은 보상 오브젝트에 다른 물리 반응으로 구분한다.
   - 증가: 부드러운 주입, 대량: 큰 출렁임/번쩍임, 감소: 누수/연기/흔들림, 0: 꺼짐/비움, 즉시완성: 발사/개방, 특별: 무지개/전설 변환.
   - 목적: 글을 많이 읽지 않아도 이번 이벤트의 성격을 알게 한다.

5. **결과 전 측정/판정**
   - 최종 결과를 바로 공개하지 말고, 보상량을 재거나 등급 트랙을 스캔하는 짧은 측정 연출을 둔다.
   - 예: 연료 게이지 측정, 행성 칩 순차 스캔, 상자 등급 판정 빛, 농장 수확량 저울.
   - 목적: "이번엔 어디까지 갔을까?" 기대감을 만든다.

6. **재도전 동기**
   - 실패/낮은 등급도 빈손처럼 보이지 않게 작은 변화는 남긴다.
   - 예: 수성까지는 못 갔어도 불씨가 남음, 작은 상자는 열림, 농장 일부 수확.
   - 목적: 다음 판을 눌러 볼 이유를 남긴다.

## 효과를 주면 안 되는 위치

- 선택지 위를 지나가거나 선택지를 흔들어 읽기 어렵게 만드는 효과.
- 문제 화면에 보상 상태판, 긴 설명, 이벤트 안내를 계속 펼쳐 두는 효과.
- 정답을 누르기 전에 답을 암시하는 효과.
- 중심 보상과 무관한 별, 코인, 하트, 폭죽을 별도 보상처럼 추가하는 효과.
- 한 화면에서 2개 이상의 큰 효과가 동시에 주인공이 되는 연출.

## 구현 패턴

- 효과는 기존 5화면 구조를 유지한 채 CSS 클래스와 짧은 JS 헬퍼로 붙인다.
- 권장 헬퍼 이름:
  - `triggerStepEffect(step, revealed)`
  - `triggerRewardTransfer(result)`
  - `triggerRewardImpact(result)`
  - `setRewardTier(value)`
  - `animateResultMeasure(targetValue)`
- `renderStatus()`는 숫자 상태를 CSS 변수 또는 `data-*` 속성으로 넘긴다. 예: `--charge`, `data-fuel-tier`.
- `finishProblem()`은 보상 모달을 띄우기 전에 이동/충격 효과를 실행한다.
- `showResult()`는 결과 문구를 바로 공개하지 말고 측정 효과가 끝난 뒤 노출한다.
- 효과 클래스는 `setTimeout` 또는 `animationend`로 제거해 다음 문제에 남지 않게 한다.
- `prefers-reduced-motion: reduce`에서는 이동·흔들림을 줄이고 색 변화 또는 opacity 변화만 남긴다.

## 차시 소재별 예시

| 소재 | 중심 변화 | 단계 효과 | 랜덤 보상 효과 | 결과 측정 |
| --- | --- | --- | --- | --- |
| 로켓/연료 | 액체가 차오르고 찰랑임 | 세로셈 칸 스파크 | 주입/누수/폭발/무지개 연료 | 연료 게이지와 행성 스캔 |
| 상자 | 상자 광택·크기·잠금 장치 변화 | 정답 칸 체크 | 상자 흔들림·빛·찌그러짐 | 등급 상자 개봉 |
| 농장/수확 | 바구니나 밭이 채워짐 | 자리 바구니 체크 | 씨앗 비·벌레·대풍작 | 수확량 저울 |
| 엘리베이터 | 층 불빛이 켜짐 | 현재 자리 층 버튼 펄스 | 급상승·하강·정전·비밀층 | 도착 층 스캔 |
| 별/우주 | 별이 모이고 궤적이 생김 | 나머지 칸 반짝임 | 별똥별·블랙홀·성운 | 별자리 완성 |
| 자물쇠/검산 | 핀이 맞고 문이 열림 | 식 칸 체결 | 금고 열림·경보·특수 열쇠 | 금고 등급 판정 |
| 분수/피자 | 조각이 맞물리고 채워짐 | 선택 조각 테두리 | 토핑 추가·조각 빠짐 | 완성 피자/접시 판정 |

## 효과 검증 질문

- 학생이 효과 때문에 문제나 선택지를 덜 보게 되지 않는가?
- 효과가 방금 한 수학 행동과 직접 연결되어 보이는가?
- 보상은 여전히 한 종류만 주인공인가?
- 좋은 이벤트와 나쁜 이벤트가 글 없이도 구분되는가?
- 결과 공개 전 기대감이 생기는가?
- 낮은 결과에도 다음 판을 해볼 작은 변화가 남는가?
