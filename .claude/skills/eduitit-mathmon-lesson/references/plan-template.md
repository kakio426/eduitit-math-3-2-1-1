# 차시 PLAN.md 템플릿

새 차시 폴더에 가장 먼저 쓰는 상세 계획. 아래 골격을 채운다.

```markdown
# <게임명(매스몬으로 시작)> 제작 계획 (<학년-학기-단원-차시>)

## Context (왜 만드나)
- 단원/차시, 영역, 성취기준(안내용 코드 + 원문 대조 필요 표시)
- 배움주제 한 줄, 이 차시가 단원에서 갖는 위치(기초/심화/종합)
- 한 줄 학습 목표(학생용)

## 수학 설계
- 문제 유형과 범위(예: 내림 있는 (두 자리)÷(한 자리))
- 문제 생성기 규칙(난수 범위, 제외 조건, 한 판 문제 수=보통 10)
- 메타 필드(top/divisor/quotient/remainder/partial 등 계산 단계에 필요한 값)
- 이 차시의 대표 오개념(= 단계 보기에 항상 넣을 오답)

## 단계 선택 설계 (핵심 메커니즘)
- N단계로 분해(각 단계: 무엇을 고르나 / 정답 / 대표 오답)
- 마지막 단계의 자동 완성 연출

## 보상/랜덤 엔진 (라벨만 교체)
- 효과 타입별 차시 라벨: 증가(+/×) · 감소(−/÷) · 0 · 즉시완성 · 특별(최고 직행)
- 중간 오답 → "결함/누수" 처리(일부러 틀리기 차단)

## 효과 설계 (references/effect-design.md 대조)
- 단계 정답 직후: 어떤 칸/블록/부품이 짧게 반응하는가
- 문제 완료 후: 계산 결과가 중심 보상으로 어떻게 이동하는가
- 중심 보상 오브젝트: 어떤 물질감(차오름/쌓임/켜짐/열림/붙음)으로 변하는가
- 랜덤 이벤트 충격: 증가/대량/감소/0/즉시완성/특별이 글 없이 어떻게 구분되는가
- 결과 전 측정: 최종 등급을 바로 공개하지 않고 무엇을 재거나 스캔하는가

## 결과 등급 (차시 자체 완결형 — 도감 아님)
- 5단계 등급 이름 + 무지개(전설), 정답 수 게이트
- "다음엔 더 멀리" 한 줄(이번 도달 → 다음 목표)

## 화면 (몇 개·무엇)
- 첫 화면 / 설명 / 문제 / 보상 / 결과 (+ 선택: 이미지 받기 카드)
- 각 화면에서 가장 큰 요소, 배지 위치
- 문제 화면 과밀 방지: 큰 문제 / 현재 단계 조작판 / 한 줄 지시문 / 선택지 외에 무엇을 숨길지 명시

## 생성 이미지 목록
표로 작성: `| 파일명(예) | 용도 | 종횡비 | 묘사 |` (커버·문제배경·보상오브젝트·등급 5~6장·결과무대)
- PNG 원본 보관 + WebP 배포 표기

## 재사용 엔진
- 복제 기준 파일, 바꾸는 함수(문제생성/단계/보상라벨/등급), 그대로 쓰는 함수

## 검증
- references/verification.md 항목 체크

## 범위 밖
```

## 작성 원칙 (앞으로도 모든 PLAN이 지킬 것)

이 습관들은 2단원 4개 PLAN을 쓰며 자리잡은 기준이다. 새 PLAN도 동일하게 지킨다.

- **구체 숫자 예시 필수**: 수학 설계·단계 선택은 추상 설명만 쓰지 말고 실제 수 예시를 함께 적는다(예 `48÷4=12`, `9×5+2=47`, 단계마다 그 단계의 계산 수치).
- **단계마다 정답 + 대표 오답을 명시**: 각 단계 보기에 정답과 "그 차시 핵심 오개념" 오답을 무엇으로 넣을지 PLAN에 적는다. 마지막 단계는 앞 단계 결과를 합쳐 최종 답을 자동 완성하는 연출로 끝낸다.
- **보상은 효과 타입 유지·라벨만 교체**: 증가/감소/0/즉시완성/특별(최고 직행) 다섯 타입을 차시 소재 이름으로만 바꾼다. 새 타입을 늘리지 않는다.
- **효과는 행동과 보상을 연결**: 단계 정답, 보상 이동, 중심 오브젝트 변화, 랜덤 이벤트 충격, 결과 측정 중 최소 3곳을 계획한다. 장식용 별·코인·하트처럼 별도 보상으로 보이면 빼고, `references/effect-design.md`를 대조한다.
- **"다음엔 더 멀리"는 구체 문구로**: 추상 표현 대신 "이번엔 텃밭까지! 다음엔 대농장!"처럼 이번 도달→다음 목표를 실제 등급명으로 적는다.
- **단원 정점(종합) 차시 표시**: 단원을 종합하는 차시는 제목 줄에 `★ 단원 정점`을 달고, 이전 차시·이전 단원에서 배운 것을 어떻게 한 흐름에 소환하는지 명시한다.
- **화면 수와 이미지 목록은 숫자로**: 화면 개수(보통 5 + 선택)와 생성 이미지 개수를 명확히 적어 빌드 전 산출물을 가시화한다.
- **문제 화면 3초 규칙을 PLAN에 적기**: 문제 화면에서 학생이 3초 안에 말해야 할 행동을 한 문장으로 적는다(예 `7과 곱한 값을 고른다`). 그 행동에 직접 필요 없는 설명판·힌트판·미리보기판은 기본 화면에서 제외한다.
- **기본 문제 화면 요소는 4개만**: 큰 문제, 현재 단계 조작판, 한 줄 지시문, 선택지만 기본 노출한다. 이전 단계는 완료 칩, 다음 단계는 `?` 또는 잠금, 힌트는 닫힌 버튼으로 계획한다.
- **검증에 차시 특화 항목 추가**: 공통 `references/verification.md` 외에, 그 차시의 대표 오개념 오답이 보기에 항상 등장하는지·수학 항등(예 `몫×나누는수+나머지=원래수`)이 성립하는지 등 차시 고유 검증을 1~2개 덧붙인다.
- **루트 `AGENTS.md`/`CLAUDE.md` 「차시 설계 점검 질문」 6개**에 모두 "그렇다"가 되는지 PLAN 단계에서 확인한다.
